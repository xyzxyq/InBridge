# Phase 9：多步骤向导

## 能力

交互配置可以在保留扁平 `controls` 的同时声明 2–8 个步骤：

```json
{
  "steps": [
    { "id": "basics", "title": "基础信息", "controlIds": ["direction"] },
    { "id": "review", "title": "确认", "controlIds": ["note"] }
  ]
}
```

没有 `steps` 的配置继续使用单页布局。向导不会改变 `version: "1"` 结果协议。

## 约束

- 步骤标识必须唯一。
- 每步至少包含一个控件。
- 每个控件必须且只能归入一个步骤。
- 展开步骤中的控件后，顺序必须与 `controls` 完全一致。
- 条件控件只能引用更早的控件，不能依赖后续步骤。
- 服务端拒绝未知、遗漏、重复或乱序引用。

## Widget 行为

- 顶部显示当前步骤、总步骤数以及完整步骤进度。
- “下一步”只校验当前步骤中可见的必填字段。
- “上一步”不重新校验，已输入值保持不变。
- 条件隐藏字段不参与校验、摘要或最终结果。
- 摘要仅在最后一步显示。
- 只有最终确认才同步模型上下文并发送 follow-up message。
- 取消、重复提交保护、重试和 JSON 复制回退沿用既有协议。

## 内置实验模板

`experiment_config` 默认生成三个步骤：

1. 基础信息：研究方向、实验环境、信息密度。
2. 训练配置：训练预算、随机种子、图表主色、补充说明。
3. 消融与确认：消融开关、条件显示的消融变量、完整摘要。

## ChatGPT 验收提示词

```text
请先调用 InBridge 的 list_interaction_templates，然后只调用
render_interaction_template，使用 experiment_config 模板生成实验配置向导。
默认研究方向为 MARL、环境为 CartPole、训练预算 80、随机种子 8，
开启消融实验。调用后等待我完成全部步骤，不要改用 render_interaction。
```

验收时应确认：出现三个步骤；返回上一步不丢值；关闭消融后相关变量隐藏；最终只提交一次；ChatGPT 能准确复述结构化结果并继续任务。
