# Phase 8：条件控件设计契约

## 目标

让一次性交互表单根据用户已经填写的前置字段显示或隐藏后续字段，同时保持 InBridge 的声明式、安全和可预测边界。

首个生产场景：实验配置模板仅在“消融实验”开启时显示“消融变量”。

## 声明格式

每个控件最多包含一个 `visibleWhen` 条件：

```json
{
  "id": "ablation_variables",
  "type": "checkbox_group",
  "label": "消融变量",
  "visibleWhen": {
    "controlId": "ablation",
    "operator": "equals",
    "value": true
  }
}
```

允许的运算符：

- `equals` / `not_equals`：用于字符串、数字和布尔值控件。
- `includes` / `not_includes`：仅用于 `checkbox_group`。

不接受脚本、表达式、嵌套逻辑、CSS 选择器或动态函数。

## 安全与确定性

- 条件只能引用当前控件之前的控件，禁止向后引用、自引用和循环依赖。
- 条件值必须与来源控件的值类型一致。
- 单选、下拉和多选条件值必须存在于来源控件的候选项中。
- 来源控件被隐藏时，依赖链上的所有后续控件也隐藏。
- 服务端完成所有规则校验，Widget 只执行已验证的白名单条件。

## UI 行为

- 控件变化后立即更新依赖控件和摘要，不触发模型调用。
- 隐藏控件不占布局空间，并设置 `aria-hidden="true"`。
- 隐藏前的本地输入暂时保留；重新显示时恢复，减少误操作损失。
- 隐藏控件不参与必填校验，不进入确认结果，也不出现在摘要中。
- 条件切换不改变按钮布局、提交状态或错误恢复流程。

## 验收

1. `defaultAblation=true` 时，实验模板显示“消融变量”。
2. 用户关闭消融实验后，该字段立即隐藏并从摘要消失。
3. 此时提交结果不包含 `ablation_variables`。
4. 再次开启时，先前选择恢复并可以提交。
5. ChatGPT 能读取条件 schema、渲染 Widget、接收结构化结果并继续任务。
